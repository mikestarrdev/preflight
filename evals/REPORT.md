# Eval report

## 1. What the system does

Preflight checks a paid social ad (copy, a creative image, or a landing-page URL)
against Meta's advertising policies and returns findings, each citing the exact policy
clause the ad breaks, quoted verbatim from the source. It runs a multi-step agent loop
(classify -> retrieve -> adjudicate -> verify -> rewrite) over a hybrid retrieval index of
the policy corpus. This report measures how well it does that, honestly enough that a
reader who has never seen the repo can decide whether to believe the numbers.

## 2. What we measure and why

Five metrics, each mapped to a user-facing failure mode.

| Metric | Measures | Failure mode it guards |
|---|---|---|
| Recall | Of ads that break a rule, the share where a finding cites an expected clause at violation/risk | A missed violation means the ad gets rejected by Meta anyway; the tool added nothing |
| False positive rate | Of clean ads, the share that produced a violation finding | A false flag costs the user a rewrite of a legal ad: worse than useless, actively expensive |
| Citation accuracy | Of emitted findings, the share whose `clause_quote` verifies as a verbatim substring of the cited chunk | A citation that does not verify is a hallucination: the trust-destroying failure |
| Explanation grounding | Of violation/risk findings, the share whose explanation quotes a verbatim `offending_span` of the ad | The explanation is what the user reads; misstating the ad ("$100" for a "$500" ad) erodes trust even when the citation is right |
| Rewrite quality | LLM-judge score (1-5) that the rewrite removes the violation while preserving marketing intent | Below ~4 the rewrite is compliant but commercially unusable |

**Four of the five are deterministic** and computed in code: recall, false-positive rate,
citation accuracy, and explanation grounding all reduce to substring checks and set
membership over the run output. Only **rewrite quality uses an LLM judge**, because
"preserves marketing intent" genuinely cannot be checked in code. The judge runs on a
pinned model (`claude-sonnet-4-6`) at temperature 0, and its prompt lives in
`evals/scorers/rewrite-judge-prompt.md` so changes to it show up in git history.

## 3. Targets

Set before running anything, so the numbers are judged against a standard rather than the
standard being drawn around the numbers. Targets apply to the tiers that measure
generalization (paraphrased, realistic), not the leaked floor (verbatim).

| Metric | Target | Rationale |
|---|---|---|
| Recall | >= 0.80 | A missed violation provides no value over Meta's own rejection |
| False positive rate | <= 0.05 | A false flag costs the user a rewrite of a legal ad |
| Citation accuracy | >= 0.98 | A citation that does not verify is a hallucination |
| Explanation grounding | >= 0.90 | Softer than citation because a slightly loose explanation is less damaging than a bad citation, but still the thing the user reads |
| Rewrite quality | >= 4.0 / 5 | Below this the rewrite is commercially unusable |

Missing a target to protect another (e.g. sacrificing recall to hold the false-positive
rate down) is a defensible product decision and is called out where it happens. Quietly
moving a target is not.

## 4. Dataset construction

Four tiers, each in its own file under `evals/dataset/`, each scored and reported
separately. They are never merged into one headline number, because they measure different
things and a blended figure would hide movement in any one of them.

| Tier | File | Cases | What it measures |
|---|---|---|---|
| 1 Verbatim | `verbatim.jsonl` | 44 | Regression floor. Leaked by construction |
| 2 Paraphrased | `paraphrased.jsonl` | 40 | Generalization on reworded violations + near-miss hard negatives |
| 3 Realistic | `realistic.jsonl` | 20 (+ clean ads pending) | Full-length ads, violation embedded in compliant text |
| 4 Images | `images.jsonl` | 21 | Creatives, including cases outside the flag taxonomy |

**Tier 1 is leaked and we say so.** Each input is a verbatim ✅/❌ example line lifted
straight from a Meta policy page, so retrieval finds the near-identical chunk trivially.
It is kept as a regression check and as the floor: if a change breaks these, it broke
something basic. It is generated by `pnpm eval:seed` from the corpus itself, so it cannot
be quietly tuned. It is **not** used to claim the system generalizes.

**Tier 2 measures generalization.** For each violating attribute in Tier 1 we wrote two
paraphrases that break the same rule (`personal-attributes:2.1`) without reusing its
wording, plus hard negatives: paraphrases of the *compliant* examples that stay compliant.
The personal-attributes policy turns on subtle distinctions ("Meet Latino professionals"
is fine, "Meet other Latino singles" is not), and those near-misses are where false
positives come from. `pnpm eval:leakage` checks that no case shares a 4+ word n-gram with
any corpus chunk; the tier is clean by that check. Labels still need a human read: the
notes field on each case records why it does or does not trip the rule, for verification.

**Tier 3 is realistic ad copy**, 66-78 words each, with a hook, body, and CTA, where the
violating span is embedded among compliant text rather than standing alone. The 20
violation cases span the verticals a marketer actually gets flagged in (weight loss,
supplements, financial services, debt relief), each labeled against a verified rule clause.
The clean half comes from Meta's Ad Library (currently-running, therefore approved, ads in
the same verticals), stored with source URL and retrieval date; those are the genuine
ground truth for the false-positive rate, since a human never wrote them to be a test case.
Creatives are not stored.

**Tier 4 expands the images** from 8 to 21, deliberately including cases *not* built around
the system's own flag taxonomy (`split_or_comparison`, `body_focus`, ...): five where the
violation lives only in rendered text (no visual flag fires), three where a flag fires but
the ad is compliant (a laundry before/after, a phone spec comparison, a Pilates class),
and clean controls. The flag-fires-but-compliant cases test whether flags are treated as
evidence or mistaken for verdicts. Fixtures are self-made HTML mockups rendered with
Playwright; no scraped creatives.

## 5. Held-out methodology

Each tier is split 70/30 into `dev` and `holdout` by stratified random sample, so both
halves cover the same policy areas and the same flag/clean balance. The split is assigned
once by `pnpm eval:split` and frozen into the data files; adding cases later (the pending
Ad Library ads) only assigns the new ones, so the held-out set never moves under us.

- Iteration happens against **dev only**. `pnpm eval` loads dev by default.
- `pnpm eval --holdout` is run at most twice: once to establish a matched baseline, once
  at the very end. The runner refuses to load holdout cases without the flag, enforced in
  code, so it cannot leak into an iteration run by accident.

The gap between final dev and final holdout numbers is itself a result. A small gap means
the changes generalized; a large gap means we overfit dev, and reporting that is more
useful than hiding it.

### Reading the baseline

The origin is the Phase 2 measurement: recall 0.14 on the 44 verbatim cases. That was not a
detection failure. Every miss *did* flag the ad; the adjudicator cited the
`example_violating` chunk (the ❌ line whose text matches the ad) instead of the governing
rule the dataset expects. Detection was 22/22; the failure was citation targeting.

The tiered baseline localizes that failure precisely. It only hurts the two
personal-attributes tiers (verbatim 0.19, paraphrased 0.27), where the corpus stores the
rule and its ❌ examples as separate chunks and the example is the near-identical top hit.
The **realistic and image tiers already sit at 1.00 recall**, because their violations
(payday loans, debt-relief promises, cure claims) map to policy documents that are mostly
rule chunks with few examples, so the adjudicator cites the rule directly with nothing to
redirect. So 4a is not a general recall fix; it is a targeted fix for one retrieval
pathology, and the tiers make that visible instead of averaging it away.

Grounding is 0.00 across the board because the pre-4b prompt never asks for an offending
span. That is the honest "before" for 4b. Note also that the flag-fires-but-compliant image
cases (laundry before/after, phone comparison, Pilates) produced findings but **zero
violations**: the false-positive rate holds at 0.00 on the image tier, so the flags are
being read as evidence, not verdicts.

## 7. Iteration log

Each entry: hypothesis, the one thing changed, before/after on **dev**, kept or reverted.

### Iteration 1 — 4a parent-rule resolution (post-hoc redirect). KEPT.

**Hypothesis.** Recall on the personal-attributes tiers is a citation-targeting failure, not
a detection failure: the adjudicator matches the ad to the near-identical ❌ example chunk
and cites that, but the dataset expects the governing rule. If a matched example is
redirected to its governing rule (walking up `clause_path`), recall should jump on the
verbatim and paraphrased tiers and barely move on realistic/images, which already cite
rules.

**Change (one thing).** In `verify.ts`, when a verified finding cites an `example_*` chunk,
redirect the citation to the governing `rule` chunk and attach the example as
`supporting_example`. Same cached adjudications; no prompt or retrieval change. Measured by
running the same dev set with the flag off (baseline) and on.

**Result (dev).**

| Tier | Recall before | Recall after | FP | Citation |
|---|---|---|---|---|
| verbatim | 0.19 (3/16) | **1.00 (16/16)** | 0.00 | 1.00 |
| paraphrased | 0.27 (4/15) | **0.93 (14/15)** | 0.00 | 1.00 |
| realistic | 1.00 (14/14) | 1.00 (14/14) | n/a | 0.97 |
| images | 1.00 (8/8) | 1.00 (8/8) | 0.00 | 0.94 |

Exactly the predicted shape. Verbatim goes to ceiling; paraphrased clears the 0.80 target
at 0.93, which is the important number because that tier is not leaked; realistic and images
are untouched because they had nothing to redirect. False-positive rate and citation
accuracy are unchanged (redirected findings are verified by construction against the rule
chunk). Cost $0.11: the fix is post-hoc, so it reused the baseline adjudications.

The single paraphrased miss is honest and worth keeping in view: `para-criminal-1-v`
("keep tanking your background checks ... get your record cleared") was matched to a health
example and personal-attributes:2.1 was judged `clear`, not a violation. The redirect can
only fix a citation the adjudicator was going to make; it cannot make the adjudicator see a
rule it dismissed.

### Iteration 2 — 4b explanation grounding (offending_span). KEPT.

**Hypothesis.** The adjudicator sometimes misstates literal values from the ad in the
explanation ("$100" for a "$500" ad). If the prompt is required to quote the offending span
verbatim and that span is verified against the input in code (the discipline `clause_quote`
already gets), the misstatements can be caught and stripped, and the grounding rate becomes
measurable.

**Change (one thing).** Added `offending_span` to the adjudicator output and prompt,
required for violation/risk findings; `verify.ts` keeps it only if it is a verbatim
substring of the input, and `explanation-grounding.ts` reports the rate. This changed the
prompt and so invalidated every cached adjudication: a full cold dev re-run.

**Result (dev).**

| Tier | Grounding before | Grounding after | Recall | FP | Citation |
|---|---|---|---|---|---|
| verbatim | 0.00 | 1.00 (27/27) | 1.00 | 0.00 | 1.00 |
| paraphrased | 0.00 | 1.00 (37/37) | 1.00 (was 0.93) | 0.00 | 0.98 |
| realistic | 0.00 | 0.89 (77/87) | 1.00 | n/a | 0.98 |
| images | 0.00 | 1.00 (45/45) | 1.00 | 0.00 | 0.99 |

Grounding goes to 1.00 on three tiers and recall, FP, and citation accuracy hold. Two
things stand out.

First, the re-adjudication under the new prompt also lifted paraphrased recall from 0.93 to
1.00: asking the model to name the exact offending words made it engage with
`para-criminal-1-v` instead of dismissing it. A prompt aimed at grounding incidentally
helped detection. Recorded because it is the truth of what happened, not attributed to a
change that was not about recall.

Second, realistic grounding is **0.89, under the 0.90 target**. Broken out by severity, the
picture is clean: **violation grounding is 42/42 = 1.00**, and 100% on every other tier too.
All ten ungrounded findings are `risk` verdicts about context that is not in the ad text at
all: age-targeting, licensing, and Special Ad Category obligations ("whether the advertiser
holds a license cannot be confirmed from the ad copy"). There is no span to quote because
the concern is not in the words. Scoping the grounding metric to violations would show 1.00
and clear the target, but that is moving the goalpost: the honest number is the stricter one,
reported with the reason it is not higher. The "$100 for a $500 ad" failure the fix targets
is text-based, and on text-based verdicts (violations) grounding is perfect.

### Iteration 3 — retrieval-side weighting, the alternative to 4a. Compared, NOT kept.

The spec asks for the citation-targeting problem to be attacked a second way and the two
compared. Iteration 1 fixed it *after* adjudication (redirect the example to its rule).
This fixes it *before*: `RETRIEVAL_RULE_BOOST=1` pulls each matched example's governing rule
into the retrieved set and lifts rule chunks above their examples, so the adjudicator has the
rule in front of it and cites it directly. Run with the redirect off, on the
personal-attributes subset (verbatim + paraphrased dev, 58 cases), same 4b prompt.

| Approach | Verbatim recall | Paraphrased recall | Citation | Violation findings (verbatim) |
|---|---|---|---|---|
| 4a redirect (post-hoc) | 1.00 | 1.00 | 1.00 | 18 |
| Retrieval boost (pre-hoc) | 1.00 | 1.00 | 1.00 | 27 |

**They tie on recall — both reach ceiling — and neither raises the false-positive rate off
0.00.** The differences are qualitative:

- The boost lets the model quote the *specific* rule sentence it is relying on; the redirect
  substitutes the whole rule chunk as the quote, which verifies but is coarser. Point to the
  boost.
- The boost also produced more decisive severities (27 violation findings vs 18 on the same
  30 verbatim cases, so more of the personal-attributes cases were called violations rather
  than risks) without any new false positives. Point to the boost.
- The redirect is post-hoc and free to evaluate ($0.11, it reused cached adjudications),
  while the boost changes the retrieved set and forces a full re-adjudication ($1.15 for the
  subset). It also works when retrieval never surfaces the rule at all, whereas the boost
  can only lift a rule that the example pointed to. Point to the redirect.

**Kept: the redirect.** On the measured targets the two are a tie, so the tie-breaker is
engineering cost and robustness, and the redirect wins both: it leaves the retrieval step
untouched, adds no per-run cost, and cannot be defeated by a retrieval miss. The boost's
advantages (precise quotes, sharper severities) are real but unmeasured by the targets; the
honest summary is that a production system would likely run both — boost for quote quality,
redirect as the safety net — and that recall, the thing under test, was already solved by the
cheaper of the two.

### Dev vs holdout: the gap analysis

This is the number the whole held-out method exists to produce, and it is mostly reassuring
with one honest exception.

**Recall, false-positive rate, and grounding show no gap.** Recall is 1.00 on both splits on
every tier; the 0.19 -> 1.00 and 0.27 -> 1.00 gains from 4a hold exactly on cases that were
never used to build the fix. The false-positive rate is 0.00 on both. Grounding is 1.00 on
both for the copy and image tiers, and the realistic combined figure is actually a hair
higher on holdout (0.91) than dev (0.89), well within small-n noise. These changes
generalized; there is nothing hidden in the gap.

**Citation dips slightly on holdout** (realistic 0.98 -> 0.94, paraphrased 0.98 -> 0.96),
which is three or four dropped quotes on ~50 findings. The verification is working as
designed (bad quotes are dropped, not emitted), and the holdout sets are small enough that a
few drops move the rate a couple of points. Not a generalization failure.

**Rewrite quality is where the gap is real: holdout verbatim 3.43 (n=7) and realistic 3.67
(n=15) fall below the 4.0 target, while dev clears it (4.33, 4.50).** The honest reading:
this is *not* overfitting, because the rewrite step was never touched in Phase 4 (it is the
Phase 2 prompt), so there was nothing to overfit to dev. It is that the ads sampled into
holdout are harder to rewrite well, amplified by small n. But a missed target is a missed
target: on held-out data the rewrites are, on those two tiers, below the bar for commercial
usability, and improving the rewrite step is the clearest next piece of work. Reporting a
0.90-point rewrite gap is more useful than the recall win alone would be, because it says
where the system is actually weak.

## 9. Known limitations and what's next

- **Tier 3's false-positive rate is not yet measured.** The 20 realistic violation cases are
  in; the 10 clean Ad Library ads (the genuine ground truth, since a human never wrote them
  as test cases) are pending and slot in without moving the split. Until then the
  false-positive rate rests on the verbatim, paraphrased, and image clean cases, all 0.00.
- **Tier 2 and 3 labels are authored-then-verified, not independently sourced.** The leakage
  check (`pnpm eval:leakage`, clean) rules out n-gram overlap, and each case carries a note
  with its rule rationale, but a second human read of those labels is the remaining step. A
  dataset where the model wrote both the question and the answer measures less than one where
  it did not.
- **Grounding on visual-only violations is soft by nature.** For an image whose violation is
  compositional (a before/after body shot with no text), there is no verbatim ad-text span to
  quote; grounding there is against the vision step's serialization, not raw copy. The metric
  is most meaningful on text-bearing elements.
- **Personal-attributes paraphrases come back as `risk`, not `violation`.** Recall credits
  both, but it means those findings get no rewrite (rewrite runs on violations), so rewrite
  quality on the paraphrased tier is measured on fewer cases than the tier's size suggests.
- **Holdout tiers are small** (6-13 cases each), so a single case swings a tier's number by
  0.08-0.17. The dev numbers carry the weight; holdout is a generalization check, not a
  precise measurement.
- **Corpus is Meta-only.** Every chunk carries a `platform` field and retrieval filters on it,
  so other platforms are an ingest, not a refactor, but they are not in this corpus.

With more time: source the Ad Library clean ads and get Tier 3's false-positive number; a
human label-verification pass on Tiers 2-3; and either exempt non-textual `risk` findings
from the grounding denominator or tag them, so the grounding metric measures only what it can
fairly measure.

$12. Cumulative spend:

**Total: $7.31** against a $12 budget. The single most expensive run ($2.39, the 4b full

keep: iteration 1 cost $0.11 because a post-hoc change reuses adjudications, and the verbatim
tier was free on every run after Phase 2 until 4b changed the prompt. Iterating without the
cache would have multiplied the cost of the four full re-runs several times over.
